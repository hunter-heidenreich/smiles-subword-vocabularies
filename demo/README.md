# SMILES subword-vocabulary demo

An interactive view of the paper ("Where to cut, how deep: BPE and Unigram-LM on
chemistry SMILES"). For a molecule, it aligns both arms on the shared glyph
stream and surfaces the paper's three contrasts live: **membership** (the two
arms learn near-disjoint multi-glyph vocabularies, small Jaccard),
**granularity** (Unigram-LM stays near-atomic and emits more tokens; BPE packs
more glyphs per token), and **compatibility** (they agree on *where* to cut but
differ in *how deeply* - the disagreement is nesting, so BPE's parse is usually a
strict coarsening of Unigram-LM's). The nesting diagram draws exactly this.

Runs on **free CPU** (no GPU, no `torch`, no `smirk`/Rust): the tokenizers are
re-implemented in pure Python and validated byte-faithful against smirk.

## What's here

| file | role |
|------|------|
| `segmenter.py` | Pure-Python, stdlib-only reimplementation of smirk's SMILES tokenization, reconstructed from a saved `tokenizer.json`. BPE segmentation is byte-for-byte exact vs smirk; Unigram matches ≈99.99% (rare Viterbi score-ties in long homopolymer chains, documented in the source). |
| `analysis.py` | Loads a cell pair, aligns both arms on the glyph stream, renders the nesting diagram + the three-contrast stats. |
| `overlay.py` | Paints a molecule's atoms by the token covering them (per arm), reusing the paper's atom-token logic; the granularity view from the graphical abstract. Local Gradio app only (needs RDKit). |
| `app.py` | The local Gradio UI (nesting diagram + RDKit molecule overlay). |
| `web/` | The static browser build: `index.html` runs `segmenter.py` + `analysis.py` in-browser via Pyodide (no server); `README.md` carries the static-Space config. |
| `build_web.py` | Assembles `web/` + the two Python modules + `tokenizers/` into one static folder to serve or upload. |
| `tokenizers/` | The shipped cell bundle - only `tokenizer.json` + `meta.yaml` per cell (~3 MB total). Generated by `export_cells.py`. |
| `export_cells.py` | **(dev)** Copies the needed cells out of `artifacts/tokenizer/` into `tokenizers/`. Needs the project artifacts. |
| `validate.py` | **(dev)** Certifies `segmenter.py` against real smirk over thousands of held-out SMILES per cell. Needs the full project env (smirk + artifacts + `data/processed/`). |

`export_cells.py` and `validate.py` are the only pieces that touch smirk or the
artifact tree; they are **not** needed to run or deploy the demo.

## Run locally

The demo's own deps (`gradio`, `rdkit`) are intentionally separate from the
research package. Easiest is a throwaway environment via `uv`:

```bash
uv run --with gradio --with rdkit python demo/app.py
# open http://127.0.0.1:7860
```

or with a plain venv:

```bash
python -m venv .demo-venv && . .demo-venv/bin/activate
pip install -r demo/requirements.txt
python demo/app.py
```

RDKit is optional - without it the app still runs, just without the 2D structure
depiction.

## Deploy for free (static, in-browser)

Hosted Gradio Spaces now require a paid plan, but the segmenter is pure-Python
stdlib, so the whole demo runs client-side under [Pyodide](https://pyodide.org)
as a **static** page - free on a Hugging Face static Space or GitHub Pages, no
server. The molecule overlay (RDKit) is the one piece that does not port; the
static build ships the nesting diagram + stats.

```bash
uv run python demo/build_web.py /tmp/site   # assemble the standalone folder
```

Then upload `/tmp/site` to a static host. For a Hugging Face static Space:

```python
from huggingface_hub import HfApi
api = HfApi()
repo = f"{api.whoami()['name']}/smiles-subword-vocabularies"
api.create_repo(repo, repo_type="space", space_sdk="static", exist_ok=True)
api.upload_folder(folder_path="/tmp/site", repo_id=repo, repo_type="space")
```

Live: <https://huggingface.co/spaces/hheiden/smiles-subword-vocabularies>

## Re-generating / re-validating (maintainers)

From the full project environment (smirk installed, `artifacts/tokenizer/`
populated):

```bash
uv run python demo/export_cells.py   # refresh demo/tokenizers/ from artifacts
uv run python demo/validate.py       # certify the pure-Python segmenter vs smirk
```

`validate.py` exits nonzero if BPE is ever inexact or Unigram exceeds a 0.1%
tie-edge tolerance.
