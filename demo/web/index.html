Where to cut, how deep: BPE vs Unigram-LM on SMILES

Two tokenizers trained on the same molecules, the same OpenSMILES glyph base, and the same target vocabulary size learn near-disjoint multi-glyph vocabularies (the Jaccard below). Yet they largely agree on where to cut and differ in how deeply: Unigram-LM stays near-atomic and emits more tokens, while BPE merges further along the same cut skeleton, so BPE's segmentation is usually a strict coarsening of Unigram-LM's. The diagram aligns both arms on the shared glyph stream (Unigram-LM above, BPE below); a filled box is a learned multi-glyph piece, a hollow box a single base glyph.
